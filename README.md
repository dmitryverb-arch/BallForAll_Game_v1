# BallForAll_Game
Техническое задание
Создать сцену, в которой игрок управляет шариком, катающимся по поверхности, собирает предметы и при сборе всех предметов на уровне побеждает. На экране должен быть UI, который отображает общее кол-во предметов и кол-во собранных.

Передвижение шарика должно быть сделано с помощью физики. Rigibody, Collider. Камера должна следовать за шариком.

Кол-во предметов должно настраиваться, либо через конфигу(ScriptableObject), либо через поле SerializeField в классе. Можешь написать спавн предметов на рандомных позициях, либо расставить точки по сцене, где они гарантированно будут спавниться.

При подборе предмета должен проигрываться визуальный эффект - разлетание огоньков или взрыв. Что угодно, просто хотелось бы чтобы ты потрогал компонент ParticleSystem и связал логику проигрывания партиклов с подбором.

Если ты разберёшься с паттерном MVVM для связки кол-ва предметов и ui это будет большим бустом. Потому что у нас на проекте используется именно этот подход. Для него в проект нужно импортнуть плагин UniRx. Если кратко - это подписка на изменение переменной. У тебя есть время, я бы на твоём месте изучила это.
Но можно спокойно использовать подход MVP.

Требования к реализации:
- Всё должно быть выполнено в одной сцене.
- Скрипты должны быть аккуратно структурированы - каждый отвечает за свою логику. Для передвижения один скрипт, для подбора предмета другой, для сущности предмета ещё один и тд.
- Без магических чисел (используй SerializeField);
- Сцену можно оформить любыми простыми примитивами (Cube, Sphere, Plane).


Реализация
Постарался максимально декомпозировать элементы, не включая ненужную логику в один объект элемент(например подсчет очков в player controller)

В проекте использовалось UniRx, рандомный спавн префабов через указание их точного количества(в следующих ветках можно поиграться с областью размещения точек), динамическая камера(следует за шариком, поворачивает влево-вправо), а также легкий партикл при сборе префаба.
